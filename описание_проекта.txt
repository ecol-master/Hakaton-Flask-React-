Парсер

1) База данных
    (у каждой таблицы будет уникальный индефикатор записи id )
    1. Таблица новостей (News)
        поля:
            title - название статьи
            text - текс статьи
            date_publish - дата публикации статьи (по умолчанию будет пустая строка если не будет известно )
            url - ссылка на конкретную статью в интернете
            key_words - список айдишников ключевых слов которые можно встретить в данной статье
            companies - список айдишников компаний которые можно встретить в данной статье
    
    2. Таблица с ключевыми словами
        поля:
            word - само слово для поиска (в именительном падеже )
    
    3. Таблица с предприятиями:
        поля:
            company - название компании (в именительном падеже )


2) Парсер
    будет асинхронная функция которая вызовет другие функции 


    1) для каждой функции свой файл (для парсинга статьи парсинг которой не будет предусмотрен )
        функция будет отвечать за парсинг по конкретному ресурсу (сайту)


    2) в каждом файле будут прописаны допустимые слова (key_words) которые можно вставлять в поиск по сайту и выбирать статьи


3) Фронтенд


4) Апишка для получения информации из базы данных
